| Google Drive | GitHub equivalent |

| Conversation folder | `conversation/` |

| O doc (`.txt` via Zapier newtxtfile) | `conversation/YYYY-MM-DD_HH-MM_O_Description.md` |

| State file | `README.md` (always reflects current package version + approval status) |

## The `_read` rule on GitHub

When G has reviewed an O doc:

1. O pushes a **new file** at `conversation/read/YYYY-MM-DD_HH-MM_O_Description_read.md` with the same content
2. O **deletes** the original `conversation/YYYY-MM-DD_HH-MM_O_Description.md`
3. At all times, `conversation/` contains only the **current active** O doc
4. `conversation/read/` is the permanent archive of all processed docs

This is identical to the Google Drive pattern: rename with `_read`, then move.
